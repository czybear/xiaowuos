# xiaowuOS V0.1 公众号文章创作 SOP

## 一、内容主线

澄木老师公众号内容五大主线：

1. **教育咨询** - 教育政策解读、升学规划、家庭教育建议
2. **人工智能教育** - AI 在教育中的应用、AI 工具介绍、智能学习法
3. **少儿编程** - 编程思维培养、课程分享、编程项目展示
4. **修心成长** - 个人成长感悟、心法分享、反思日记
5. **科创纪事** - 科技创新动态、创客活动记录、项目实战故事

## 二、文章标准结构

### 标准三段式结构：

1. **开头（15-20%）** - 故事引入，建立共鸣
2. **中段（60-70%）** - 案例分析，深度讲解
3. **结尾（15-20%）** - 金句总结，行动号召

### 篇幅建议：
- 常规文章：1500-2500 字
- 深度文章：3000-5000 字
- 轻量分享：800-1200 字

## 三、标题生成规则

### 四种标题类型：

1. **疑问式**
   - 格式："如何/为什么/什么是 + [核心问题]"
   - 示例：《如何让孩子爱上编程而不厌学？》

2. **数字式**
   - 格式："[数字] 个 + [关键词] + [价值点]"
   - 示例：《5 个 AI 工具让学习效率提升 300%》

3. **对比式**
   - 格式："[传统做法] vs [新方法]"
   - 示例：《死记硬背还是 AI 辅助？2024 学习法大对比》

4. **故事式**
   - 格式："[人物] 的 [经历]，让我明白了 [道理]"
   - 示例：《10 岁编程天才妈妈的反思：别逼孩子赢在起跑线》

### 标题避坑：
- ❌ 避免过长（超过 20 字）
- ❌ 避免过度营销词汇
- ❌ 避免与内容不符
- ✅ 确保包含核心关键词
- ✅ 能引发好奇心但不过度

## 四、开头故事法

### 故事三要素：
1. **具体人物** - 真实或典型的人物画像
2. **具体场景** - 可感知的时间、地点、情境
3. **具体冲突** - 遇到的问题或困境

### 写作模板：

```
[人物] + [场景] + [问题] + [转折] + [引出主题]

例如：
"上周六下午，李明妈妈在家长会结束后拉住我，
眼眶泛红地说：'老师，我家孩子学编程三个月了，
现在看到电脑就烦躁，我是不是做错了？'
这个问题，让我想到了今天分享的主题..."
```

### 开头要点：
- 篇幅控制在 200-300 字
- 用对话或细节描写增加真实感
- 2-3 句内必须引出文章主题
- 避免过度渲染，保持自然

## 五、中段案例法

### 案例结构：

1. **案例背景** - 说明案例的来源和背景
2. **问题分析** - 拆解问题，分析原因
3. **解决方案** - 给出具体的方法和步骤
4. **结果验证** - 效果展示或数据支持

### 三种案例类型：

1. **成功案例** - 正向示范
   - 格式：问题 + 方法 + 结果 + 关键点
   - 示例：7 岁小朋友如何从零基础到完成第一个游戏

2. **失败案例** - 逆向思考
   - 格式：问题 + 错误做法 + 改进 + 新结果
   - 示例：某机构 AI 课程的三个坑，我们如何避坑

3. **对比案例** - 方法对比
   - 格式：两种方法对比 + 优劣分析 + 适用场景
   - 示例：Scratch vs Python，不同年龄段的最佳选择

### 案例写作要点：
- 每个案例控制在 200-400 字
- 至少包含 2-3 个具体案例
- 案例之间要有逻辑递进
- 关键信息要具体（时间、数据、人名）

## 六、结尾金句法

### 金句类型：

1. **总结式** - 用一句话总结全文
   - 示例："真正的编程教育，不是教代码，而是教思考"

2. **号召式** - 引导读者行动
   - 示例："这个周末，和孩子一起打开编程软件吧"

3. **留白式** - 引发思考
   - 示例："AI 不会取代教师，但会用 AI 的教师将取代不会用 AI 的教师"

4. **呼应式** - 呼应开头故事
   - 示例："现在回头看，李明妈妈的问题，其实是所有家长的焦虑"

### 金句要点：
- 简洁有力，不超过 20 字
- 有记忆点，适合分享
- 与主题高度相关
- 避免说教，保持温和

## 七、排版规则

### 段落格式：
- 每段不超过 5 行（移动端友好）
- 段间距：1 倍行距
- 首行不缩进

### 重点标记：
- **加粗** - 关键词、重点概念
- _斜体_ - 引用、强调
- ~删除线~ - 过时观点、对比

### 特殊格式：
- 引用块：`> ` - 用于案例、引语
- 列表：`- ` 或 `1. ` - 步骤、清单
- 代码块：``` - 编程示例

### 分隔线：
- 章节间使用 `---` 或 `***` 分隔

## 八、配图建议

### 配图原则：
1. **相关性** - 与内容直接相关
2. **清晰度** - 分辨率 720px 以上
3. **简洁性** - 信息密度适中
4. **版权** - 使用原创或免费版权图

### 配图类型：
1. **封面图** - 文章主题可视化，尺寸建议 900x383px
2. **插图** - 解释概念，如流程图、对比图
3. **截图** - 软件操作、界面展示
4. **数据图** - 统计、对比数据
5. **金句图** - 结尾金句视觉化，适合分享

### 配图建议位置：
- 开头后插入 1 张封面图
- 每个案例段落配 1 张辅助图
- 金句部分配金句图
- 全文 4-6 张为宜

## 十、内容复用矩阵

### 一篇文章如何拆成：

#### 1. 短视频（60-90 秒）
- **脚本来源**：
  - 开头故事（15-20 秒）
  - 中段核心案例（30-40 秒）
  - 结尾金句（5-10 秒）
- **制作要点**：
  - 口播 + 画面配合
  - 每 3-5 秒画面切换
  - 字幕突出关键词
  - 结尾引导关注

#### 2. 海报（单图）
- **设计要素**：
  - 金句内容（主视觉）
  - 文章标题（副标题）
  - 公众号二维码
  - 澄木老师 IP 形象
- **规格**：750x1334px
- **使用场景**：朋友圈、社群分享

#### 3. 四格漫画
- **脚本拆分**：
  - 格 1：问题引入（开头故事）
  - 格 2：错误做法（反面案例）
  - 格 3：正确方法（核心方法）
  - 格 4：金句总结 + 引导
- **制作要点**：
  - 风格统一，简洁可爱
  - 每格配简短文字
  - 适合公众号底部嵌入
  - 也可独立传播

### 复用流程图：

```
公众号文章
    ├── 短视频脚本 → 拍摄剪辑 → B 站/抖音
    ├── 海报设计 → 朋友圈/社群
    └── 四格漫画 → 公众号内嵌/独立传播
```
