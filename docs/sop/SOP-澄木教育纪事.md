# 澄木教育纪事 SOP

## 一、定位口径

- 栏目正式名称统一为：`澄木教育纪事`
- 旧称 `科创教育纪事` 视为历史阶段名称；新产出、新同步、新目录命名均优先使用 `澄木教育纪事`
- 如果历史文件尚未改名，可先保留文件名，但正文标题、飞书文档标题、对外展示标题应优先更新为新名称

## 二、当前结构

- 周更结构保持：`全球 3 条 + 全国 3 条 + 上海 3 条`
- 每期保留：`本期摘要`、`全球风向`、`全国进展`、`上海落地`、`写在最后`、`参考链接`
- 文章目标不是资讯堆砌，而是：`记录变化，留下判断`

## 三、标题规范

- 推荐标题格式：`澄木教育纪事丨00X期：主题`
- 预览稿可用更灵活副标题，但正式稿统一收束为 `期数 + 冒号主题`
- 飞书正文标题、同步后的文档标题、目录展示标题，尽量保持一致，避免一个期次出现多个正式名称

## 四、排版规范

- 飞书版为当前排版微调后的优先参考版本
- Markdown 初稿负责内容结构清晰；飞书成稿负责最后一轮视觉微调
- 微调重点：
- 段落不要过长，重要判断单独成段
- `时间 / 来源类型 / 事件要点 / 大米指数 / 澄木视角 / 澄木点评` 的节奏保持稳定
- 摘要区优先短句，避免一大段说明压在开头
- 同一层级标题样式保持一致，不混用不同命名口径

## 五、生产流程

1. 先复制最新母版，默认使用：`deliverables/澄木教育纪事-周更生产母版-V1.md`
2. 先定本期主题，再补齐 `全球 / 全国 / 上海` 各 3 条，避免先堆素材后找主题
3. 完成 `澄木综评`，明确本周最重要的 1 个判断，不要写成九条资讯复述
4. 补写 `本周教学手记 / 本周教学心得 / 下周动作`，把栏目从资讯稿推进成教学周记
5. 本地完成 Markdown 审校后，再同步到飞书文档
6. 在飞书里做最后一轮标题、分段、留白、可读性微调
7. 如飞书版发生正式命名或排版规则变化，要回写本地 Markdown 与 SOP

## 六、每期必交付内容

- `本期摘要`：用 3 句分别概括 `全球 / 全国 / 上海`
- `全球风向 / 全国进展 / 上海落地`：保持 `3 + 3 + 3`
- `澄木综评`：收束为 1 个中心判断，最多拆成 3 个转向
- `本周教学手记`：写 1 个真实课堂片段，控制在 `300 - 600` 字
- `本周教学心得`：只写 1 个最值得留下的教学判断
- `对课程体系的启发`：回答“这周的变化要怎么改课”
- `下周动作`：固定 3 条，分别对应 `备课 / 观察 / 验证`
- `参考链接`：正文引用过的来源要完整回收

## 七、文件命名规则

- 正式稿：`澄木教育纪事00X期-正式版.md`
- 预览稿：`澄木教育纪事00X期-预览版.md`
- 母版：`澄木教育纪事-周更生产母版-V1.md`
- 周报化、结构改版、栏目策略类文档：`澄木教育纪事-xxx-V1.md`
- 历史文件若仍保留 `科创教育纪事` 命名，可暂不回退，但新文件统一切到 `澄木教育纪事`

## 八、同步规则

- 飞书 `docx` 成稿与本地 Markdown 应保持同主题同标题
- 如果飞书中已完成正式命名调整，应以飞书成稿为准，反向修正本地标题
- 后续同步目录中，与本栏目相关的文件夹名称建议逐步统一为：`澄木教育纪事`
- 本栏目文件完成新增或实质修订后，默认执行：
  `node automation/feishu-sync.mjs auto-sync <markdown-file>`

## 九、当前已确认变更

- 栏目名已从 `科创教育纪事` 调整为 `澄木教育纪事`
- 飞书文档已做过一轮排版微调
- 栏目已确认升级方向：保留 `全球 / 全国 / 上海` 骨架，并新增 `教学手记 / 教学心得 / 下周动作`
- 本 SOP 已同步记录上述变更，作为后续批量生产与飞书同步的统一依据
