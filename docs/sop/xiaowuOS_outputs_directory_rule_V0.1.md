## 一、正源规则（核心原则）

从本规范生效之日起：

| 产出物类型 | 正源路径 | 说明 |
|-----------|---------|------|
| **全部正式产出物** | `/home/john/xiaowuOS/outputs/` | 唯一正源，所有交付物归位至此 |
| **课程教案、课件、PPT、备课包** | `outputs/courses/` | 按 YYYYMMDD_课次名称 分目录存放 |
| **正式报告（PRD、方案、诊断）** | `outputs/reports/` | 业务化报告、系统诊断、技术方案 |
| **通用 PPT（非课程专属）** | `outputs/slides/` | 独立于课程的演示文稿 |
| **媒体产出（图片、视频、音频）** | `outputs/media/` | 标题图、视频素材、录音 |
| **导出文件** | `outputs/exports/` | 飞书导出、平台导出等 |
| **长期项目资料** | `projects/<项目名称>/` | 如零式协约、独立业务线 |

## 二、禁止混放规则

### .xiaowuOS/ — 系统运行目录，禁止存放正式交付物

| 路径 | 用途 | 禁止内容 |
|------|------|---------|
| `.xiaowuOS/bin/` | 系统脚本 | ❌ 课程文件、报告、SOP |
| `.xiaowuOS/config/` | 配置文件 | ❌ 任何非配置内容 |
| `.xiaowuOS/data/` | 运行时数据 | ❌ 交付物 |
| `.xiaowuOS/docs/` | 内部参考文档 | ❌ 正式课程教案、正式 PPT（Phase 2 已清理副本） |

### workspace/ — 临时工作区，不放正式产出物

| 子目录 | 用途 | 禁止内容 |
|--------|------|---------|
| `workspace/inbox/` | 待处理输入 | ❌ 正式发布内容 |
| `workspace/staging/` | 中间稿、规划类方案 | ❌ 定版交付物 |
| `workspace/tmp/` | 临时文件 | ❌ 任何正式内容 |

### docs/ — 系统文档，不放具体课次产出物

| 子目录 | 用途 | 禁止内容 |
|--------|------|---------|
| `docs/sop/` | 标准化作业流程 | ❌ 单次课程教案（SOP ≠ 单课教案） |
| `docs/system_audit/` | 系统审计报告 | ❌ 课程内容 |
| `docs/architecture/` | 架构说明 | ❌ 交付物 |
| `docs/decisions/` | 决策记录 | ❌ 课程产出 |

## 三、命名规范

### outputs/courses/ — 课次目录命名

```
YYYYMMDD_课次简称/
├── [日期]_[内容]_版本.md        # 教案文档
├── [日期]_[内容]_版本.pptx      # 课件 PPT
├── _archive/                   # 过程稿归档（可选）
└── V{n}/                       # 版本迭代目录（可选）
```

**示例：**
```
20260613_GESP-C4-植物大战僵尸/
├── 00_课程索引.md
└── V0.2/
    ├── GESP-C4-轩轩最后一波防守_应急提纲_V0.2.md
    ├── GESP-C4-轩轩最后一波防守_教师讲解版_V0.2.md
    ├── GESP-C4-轩轩最后一波防守_题目与代码_V0.2.md
    └── 轩轩-GESP-C4-优秀编程习惯清单_V0.2.md
```

### outputs/reports/ — 报告命名

```
[系统/项目名]-[内容类型]-[日期].md
或
[系统/项目名][内容类型]_[版本].md
```

**示例：**
```
澄木系统PRD-v0.1.md
OpenClaw-cron-诊断-2026-05-24.md
```

### outputs/articles/ — 文章命名

```
[栏目名][序号期]-[版本].md
或
[栏目名]-[日期].md
```

## 四、新产出物工作流

当生成新的正式产出物时：

1. **草稿阶段** → `workspace/staging/`（中间稿、未定版）
2. **审核通过** → `rsync -av --ignore-existing <草稿路径> outputs/<对应目录>/`
3. **正源确认** → 仅在 `outputs/` 中维护最终版本
4. **旧稿处理** → 移入课次 `_archive/` 或保留在 staging（待清理）
