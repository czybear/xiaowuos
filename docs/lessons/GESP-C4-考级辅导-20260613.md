## 二、⭐ 典型题讲解方案（15分钟核心）

### 题目：模拟程序运行结果（综合题）

```python
def mystery(n):
    a = 1
    b = 0
    while a <= n:
        b += a
        if b >= 10:
            break
        a *= 2
    return b

print(mystery(3))
```

### 讲解步骤：

**第1步（2min）— 整体观察**
- 函数接收什么？→ `n`，返回值？→ `b`
- 有循环、条件判断、乘法更新——典型模拟题

**第2步（5min）— 手写执行表**
> 要求学生自己在草稿纸上画表：

| 轮次 | a (入值) | b (入值) | a <= n? | b += a 后 b | b >= 10? | a *= 2 后 a |
|------|---------|---------|---------|------------|----------|------------|
| 初始 | - | - | - | - | - | - |
| 第1轮 | 1 | 0 | 是(1≤3) | 0+1=**1** | 否 | 2 |
| 第2轮 | 2 | 1 | 是(2≤3) | 1+2=**3** | 否 | 4 |
| 第3轮 | 4 | 3 | **否**(4>3) | 不执行 | - | - |

→ 循环结束，返回 `b = 3`

**关键提问：**
- "a *= 2"这行每轮都执行吗？→ 注意 if break 会提前退出！
- "如果 n=10 结果是多少？" → 让学生自己算一遍（答案：7）

**第3步（4min）— 提炼通用解法**
```
模拟类题目的三步走：
1. 画变量跟踪表（横轴=轮次，纵轴=变量）
2. 逐行执行，不跳步骤
3. 特别注意 break/return 的触发条件
```

**第4步（4min）— 变式训练**
> 改一数字，让思路不变但答案不同：

```python
# 变式：把 a *= 2 改为 a += 1，n=5，结果？
# 学生独立完成后对答案
# 答案：b = 0+1+2+3 = 6（第4轮 b=10>=10, break）
```

## 三、高频易错点清单（C4级别）

| # | 易错点 | 典型表现 | 考场对策 |
|---|--------|---------|---------|
| 1 | **运算符优先级混淆** | `a and b or c` vs `(a and b) or c` 算错 | 先加括号再读 |
| 2 | **while/for 边界** | off-by-one，漏算最后一轮 | 强制画执行表 |
| 3 | **列表索引越界思维** | `[-1]`取最后一个而非报错（Python特性） | C4考Python时记住这个特性 |
| 4 | **字符串和数字混用** | `"10" > 2` → TypeError，但 `"10" > "9"` → False | 类型先判断 |
| 5 | **if-elif-else 链式逻辑** | 只进第一个满足条件的分支，不会全走 | 牢记"选一路"原则 |
| 6 | **函数返回值遗漏** | 写了 print 没写 return，函数返回 None | 读完函数找 return |
| 7 | **全局变量 vs 局部变量** | 在函数内修改外部变量是否需要 global | C4常考陷阱题 |

### 考场提醒话术：
> "同学们，C4最容易丢分的不是不会做，而是**看太快**。每道程序阅读题强制自己画执行表，宁可多花30秒，也不要空脑算。"
